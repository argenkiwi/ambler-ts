# Program Specifications

This program implements a game of "20 Questions" where the human thinks of an object and the LLM (running locally via Ollama) tries to guess it by asking up to 20 yes/no questions.

## Rules
- The LLM can ask up to 20 yes/no questions.
- The LLM can make up to 3 guesses.
- A guess must begin with the words "Is the answer ".
- A guess does NOT increase the question count.
- The game ends if the LLM guesses correctly (Win).
- The game ends if the LLM makes 3 incorrect guesses (Loss).
- The game ends if the question count reaches 20 (Loss).

## Nodes

### Ollama Check
- Role — Startup node.
- Logic — Verifies if Ollama is reachable at the configured host. If not, prompts the user to enter a host URL.
- Edges — On success, transitions to `MODEL_SELECT`. On error, loops back to `OLLAMA_CHECK`.

### Model Select
- Role — Configuration node.
- Logic — Lists available models and prompts the user to select one.
- Edges — Transitions to `GAME_START`.

### Game Start
- Role — Initialization node.
- Logic — Initializes the game state, setting the question count to 0 and guess count to 0. It provides a system prompt explaining the rules (20 questions, 3 guesses). Displays a message telling the human to think of an object.
- Edges — Transitions to `LLM_QUESTION`.

### LLM Question
- Role — Question generation node.
- Logic — Sends the conversation history to the LLM. The LLM is instructed to ask a single yes/no question or make a guess. If making a guess, it must prefix it with "Is the answer ". The assistant's response is printed and added to history.
- Dynamic Hinting — If the question count reaches 15 or more, or if guesses have been made, a dynamic system hint is injected into the messages sent to the LLM, reminding it of the remaining attempts.
- Edges — Increments the question count IF it's a question, or the guess count IF it's a guess. Transitions to `HUMAN_RESPONSE`.

### Human Response
- Role — Input node.
- Logic — Prompts the human for a response (e.g., "yes", "no", "maybe", "sometimes"). If the human types "quit", it ends the game.
- Edges — Transitions to `CHECK_CONDITION`. If the human quits, transitions to `GAME_OVER`.

### Check Condition
- Role — Game logic node.
- Logic — Evaluates if the game should continue. It checks if:
  1. The human confirmed the LLM's guess (Win).
  2. The LLM has made 3 incorrect guesses (Loss).
  3. The question count has reached 20 (Loss).
  4. The LLM should ask another question.
- Edges — If Win or Loss condition met, transitions to `GAME_OVER`. Otherwise, transitions to `LLM_QUESTION`.

### Game Over
- Role — Termination node.
- Logic — Prints a final message based on the outcome (win/loss/quit).
- Termination — Exits the process.

## Shared State

- `messages`: A list of messages (history) with `role` and `content`.
- `host`: The Ollama host URL.
- `model`: The selected model name.
- `questionCount`: An integer tracking the number of questions asked by the LLM.
- `guessCount`: An integer tracking the number of guesses made by the LLM.
